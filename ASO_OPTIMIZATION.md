### 🎯 Universal Design Guidelines

#### **Background Selection Strategy:**

1. **Primary Screenshots (1-3):** Use brand gradients (Primary, Secondary, Accent)

   - Screenshot 1: Primary → Secondary gradient
   - Screenshot 2: Primary Blue gradient
   - Screenshot 3: Accent → Secondary gradient

2. **Result Screenshots (4):** Use success/achievement colors

   - Success green to primary blue gradient

3. **Trust Screenshots (5):** Use secondary/purple tones

   - Purple to blue gradient for security feel

4. **Premium Screenshots (6):** Use premium gradient
   - Secondary to accent gradient

#### **Text Overlay Guidelines:**

- **Title Font:** Bold, white or dark (depending on background)
- **Subtitle Font:** Medium weight, white with 90% opacity or dark with 80% opacity
- **Contrast:** Ensure 4.5:1 contrast ratio for accessibility
- **Position:** Top-left or bottom-left (avoid covering key UI elements)
- **Shadow:** Add subtle text shadow for readability on gradients

#### **Screenshot Composition:**

- **Padding:** 16-24px safe area around device frame
- **Device Frame:** Modern iPhone (iPhone 15 Pro) or Pixel device
- **Status Bar:** Show native iOS/Android status bar
- **Overlay Elements:** Subtle, non-intrusive (max 10% opacity)
- **Focus:** Screenshot should be 80% app UI, 20% background/overlay

#### **Platform-Specific Considerations:**

**iOS:**

- Use rounded corners matching iOS design
- Include notch/safe area indicators
- White status bar icons on dark gradients
- Dark status bar icons on light backgrounds

**Android:**

- Use Material 3 rounded corners
- Include navigation bar
- Adapt to Android's system UI colors
- Consider notch variations

#### **🚀 Implementation Tips:**
